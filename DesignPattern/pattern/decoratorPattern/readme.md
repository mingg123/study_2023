### 데코레이터 패턴
- 상속을 남용하는 사례를 살펴보고 객체 작성 형식으로 실행중에 클래스를 꾸미는데 사용함 
- 기존 클래스 코드를 바꾸지 않고 새로운 객체에 동적으로 요소를 추가할 수 있음 
- 서브클래스를 만들 때보다 훨씬 유연하게 기능을 확장할 수 있다. 


### 잘못된 예시 
1. abstraclass로 Beverage를 만들고, 디카페인이나 카페인 거기서 전부 cost 함수를 구현하도록 함 
2. 휘핑크림이 추가된 디카페인(?) 클래스가 계속 늘어남

### 데코레이터 패턴 장단점
장점
- OCP 원칙을 지킴 (기존 코드를 수정하지 않고 새로운 코드 추가 가능)
- 객체의 기능을 동적으로 변경할 수 있음 

단점
- 객체를 데코레이터로 감싸기 때문에 코드 가독성이 떨어질 수 있음 & 디버깅 어려움 
- 한눈에 파악하기 어려움 
- 중첩된 형태라 성능에 영향을 미칠 수 있음
